## 学习阶段规划

### 基础阶段（4月 - 5月）

**目标**：理解计算机硬件原理，掌握核心计算

#### 第1周：计算机系统概述

| 内容 | 学习要点 |
|------|----------|
| 发展历程 | 四代计算机 |
| 系统层次 | 硬件/软件层次结构 |
| 性能指标 | 主频、CPI、MIPS、MFLOPS |
| 冯·诺依曼结构 | 五大部件、存储程序原理 |

**性能指标公式**：
```
CPU时钟周期 = 1 / 主频
CPI = 执行程序所需时钟周期数 / 指令条数
CPU执行时间 = 指令条数 × CPI × 时钟周期
MIPS = 指令条数 / (执行时间 × 10^6)
     = 主频 / (CPI × 10^6)
```

#### 第2-3周：数据的表示与运算（重点）

| 内容 | 学习要点 |
|------|----------|
| 进制转换 | 二/八/十/十六进制互转 |
| 原码/反码/补码 | 定义、转换、表示范围 |
| 定点数运算 | 补码加减、溢出判断 |
| 浮点数 | IEEE 754标准、规格化 |
| 浮点数运算 | 对阶、尾数运算、规格化、舍入 |
| ALU | 加法器、ALU结构 |

**补码运算**：
```
[X]补 + [Y]补 = [X+Y]补
[X]补 - [Y]补 = [X]补 + [-Y]补

溢出判断：
1. 双符号位法：00正确，11正确，01正溢出，10负溢出
2. 单符号位法：Cs ⊕ C1（符号位进位 ⊕ 最高数值位进位）
```

**IEEE 754 浮点数**：
```
单精度（32位）：1位符号 + 8位阶码 + 23位尾数
  阶码偏移量：127
  表示范围：约 ±3.4×10^38

双精度（64位）：1位符号 + 11位阶码 + 52位尾数
  阶码偏移量：1023

规格化：尾数最高位隐含为1
真值 = (-1)^S × 1.M × 2^(E-偏移量)
```

**重点计算**：
- [ ] 补码加减法、溢出判断
- [ ] IEEE 754浮点数与真值互转
- [ ] 浮点数加减运算过程

#### 第4-5周：存储系统（重点）

| 内容 | 学习要点 |
|------|----------|
| 存储器层次 | Cache-主存-辅存 |
| 主存 | SRAM/DRAM、存储器扩展 |
| Cache | 映射方式、替换算法、写策略 |
| 虚拟存储器 | 页式/段式、TLB |
| 外存 | 磁盘结构、磁盘阵列 |

**存储器扩展**：
```
位扩展：增加数据位数
字扩展：增加存储容量
字位同时扩展：两者结合

需要芯片数 = (总容量/单片容量) × (数据位数/单片位数)
```

**Cache映射方式**：

| 映射方式 | 主存块号→Cache行号 | 特点 |
|----------|-------------------|------|
| 直接映射 | i = j mod 2^c | 简单，冲突多 |
| 全相联 | 任意 | 灵活，硬件复杂 |
| 组相联 | i = j mod Q（Q为组数） | 折中 |

**Cache地址结构**：
```
直接映射：| 标记 | 行号 | 块内地址 |
全相联：  | 标记 | 块内地址 |
组相联：  | 标记 | 组号 | 块内地址 |
```

**Cache性能计算**：
```
命中率 h = 命中次数 / 总访问次数
平均访问时间 = h × tc + (1-h) × tm
            = tc + (1-h) × (tm - tc)

其中 tc = Cache访问时间，tm = 主存访问时间
```

#### 第6周：指令系统

| 内容 | 学习要点 |
|------|----------|
| 指令格式 | 操作码、地址码 |
| 寻址方式 | 立即/直接/间接/寄存器/基址/变址/相对 |
| CISC与RISC | 特点对比 |

**寻址方式**：

| 寻址方式 | 有效地址EA | 特点 |
|----------|-----------|------|
| 立即寻址 | 指令中 | 快，范围小 |
| 直接寻址 | EA = A | 简单 |
| 间接寻址 | EA = (A) | 灵活，慢 |
| 寄存器寻址 | EA = Ri | 最快 |
| 寄存器间接 | EA = (Ri) | 较快 |
| 基址寻址 | EA = (BR) + A | 程序浮动 |
| 变址寻址 | EA = (IX) + A | 数组访问 |
| 相对寻址 | EA = (PC) + A | 转移指令 |

#### 第7-8周：中央处理器（重点）

| 内容 | 学习要点 |
|------|----------|
| CPU结构 | 运算器、控制器 |
| 指令执行 | 取指、译码、执行、访存、写回 |
| 数据通路 | 单总线、多总线 |
| 控制器 | 硬布线、微程序 |
| 流水线 | 概念、性能指标、冒险 |

**流水线性能**：
```
理想情况：
  流水线时钟周期 = max{各段时间}
  吞吐率 TP = n / T总（n条指令的总时间）
  加速比 S = T顺序 / T流水

实际情况需考虑：
  结构冒险：资源冲突
  数据冒险：RAW、WAR、WAW
  控制冒险：转移指令
```

**流水线计算**：
```
n条指令，k级流水，每段时间Δt：
  流水线执行时间 = (k + n - 1) × Δt
  顺序执行时间 = n × k × Δt
  加速比 = n×k / (k+n-1)
  当n→∞时，加速比→k
```

#### 第9周：总线与I/O系统

| 内容 | 学习要点 |
|------|----------|
| 总线概述 | 分类、性能指标 |
| 总线仲裁 | 集中式（链式/计数器/独立请求） |
| 总线定时 | 同步/异步/半同步 |
| I/O方式 | 程序查询/中断/DMA/通道 |
| 中断系统 | 中断处理过程、多重中断 |

**总线带宽**：
```
总线带宽 = 总线宽度 × 总线频率
        = 数据位数 × (1/总线周期)
```

**DMA传送**：
```
DMA方式特点：
1. 数据传送由DMA控制器控制
2. CPU与DMA控制器分时使用总线
3. 传送结束后DMA向CPU发中断

DMA传送方式：
- 停止CPU访存
- 周期挪用（周期窃取）
- DMA与CPU交替访存
```

## 学习方法建议

### 组成原理学习特点

1. **抽象性强**：需要画图理解
2. **计算复杂**：公式多，需要大量练习
3. **综合性强**：各章节联系紧密
4. **重实践**：结合实际硬件理解

### 计算题练习建议

```
练习重点：
1. IEEE 754浮点数转换
2. Cache地址划分与命中率
3. 存储器扩展与片选
4. 流水线时间与加速比
5. 总线带宽计算
```
