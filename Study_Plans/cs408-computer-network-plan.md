## 学习阶段规划

### 基础阶段（5月 - 6月）

**目标**：理解网络分层模型，掌握核心协议

#### 第1周：计算机网络体系结构

| 内容 | 学习要点 |
|------|----------|
| 网络概述 | 组成、分类、性能指标 |
| 体系结构 | OSI七层、TCP/IP四层 |
| 参考模型 | 各层功能、协议、设备 |

**性能指标**：
```
速率：数据传输速率，单位 bps
带宽：最高数据率
时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
  发送时延 = 数据长度 / 发送速率
  传播时延 = 信道长度 / 传播速度
时延带宽积 = 传播时延 × 带宽
往返时间 RTT
吞吐量
```

**协议分层**：

| 层次 | 功能 | 协议举例 | 设备 |
|------|------|----------|------|
| 应用层 | 应用服务 | HTTP/FTP/DNS/SMTP | - |
| 传输层 | 端到端通信 | TCP/UDP | - |
| 网络层 | 路由转发 | IP/ICMP/ARP | 路由器 |
| 数据链路层 | 帧传输 | Ethernet/PPP | 交换机/网桥 |
| 物理层 | 比特传输 | - | 集线器/中继器 |

#### 第2周：物理层

| 内容 | 学习要点 |
|------|----------|
| 通信基础 | 数据、信号、码元、波特率 |
| 信道容量 | 奈奎斯特定理、香农定理 |
| 编码与调制 | 数字/模拟信号编码 |
| 传输介质 | 双绞线、光纤、无线 |
| 物理层设备 | 中继器、集线器 |

**信道容量公式**：
```
奈奎斯特定理（无噪声）：
  C = 2W log₂V (bps)
  W = 带宽(Hz)，V = 信号电平级数

香农定理（有噪声）：
  C = W log₂(1 + S/N) (bps)
  S/N = 信噪比（需转换：S/N(dB) = 10 log₁₀(S/N)）
```

#### 第3-4周：数据链路层

| 内容 | 学习要点 |
|------|----------|
| 功能与服务 | 成帧、差错控制、流量控制 |
| 差错控制 | 奇偶校验、CRC、海明码 |
| 流量控制 | 停止等待、滑动窗口（GBN/SR） |
| 信道划分 | FDMA/TDMA/CDMA/WDM |
| 随机访问 | ALOHA、CSMA、CSMA/CD、CSMA/CA |
| 以太网 | MAC地址、帧格式、VLAN |
| 交换机 | 自学习、转发表 |

**CRC计算**：
```
1. 数据后补r个0（r为生成多项式最高次幂）
2. 模2除法除以生成多项式
3. 余数即为CRC校验码
4. 发送：数据 + CRC
```

**滑动窗口协议**：

| 协议 | 发送窗口 | 接收窗口 | 特点 |
|------|----------|----------|------|
| 停止等待 | 1 | 1 | 效率低 |
| GBN | >1 | 1 | 累积确认，出错全部重传 |
| SR | >1 | >1 | 选择重传，效率高 |

**窗口大小限制**（n位序号）：
```
GBN：W_send ≤ 2^n - 1
SR：W_send + W_recv ≤ 2^n
```

**CSMA/CD**：
```
争用期 = 2τ（τ为单程传播时延）
最小帧长 = 争用期 × 数据传输率

以太网：争用期 = 51.2μs，最小帧长 = 64字节
```

#### 第5-6周：网络层（重点）

| 内容 | 学习要点 |
|------|----------|
| 功能概述 | 路由选择、分组转发 |
| IPv4 | 地址格式、子网划分、CIDR |
| IP协议 | 首部格式、分片 |
| ICMP | 差错报告、询问报文 |
| ARP/RARP | 地址解析 |
| 路由算法 | 距离向量、链路状态 |
| 路由协议 | RIP、OSPF、BGP |
| IPv6 | 地址格式、与IPv4对比 |
| 组播 | IGMP |
| NAT | 地址转换 |

**IP地址与子网划分**：
```
IP地址 = 网络号 + 主机号

子网掩码：
  /24 表示前24位为网络号
  子网数 = 2^(借用的主机位数)
  每个子网主机数 = 2^(剩余主机位数) - 2

CIDR：
  网络地址/前缀长度
  例：192.168.1.0/24
```

**IP分片**：
```
MTU = 最大传输单元
片偏移 = 该片在原数据报中的偏移 / 8
MF = 1 表示后面还有分片
```

**路由算法对比**：

| 协议 | 类型 | 算法 | 特点 |
|------|------|------|------|
| RIP | 距离向量 | Bellman-Ford | 跳数≤15，收敛慢 |
| OSPF | 链路状态 | Dijkstra | 收敛快，开销大 |
| BGP | 路径向量 | - | 自治系统间 |

#### 第7周：传输层

| 内容 | 学习要点 |
|------|----------|
| 功能与服务 | 端到端通信、端口 |
| UDP | 特点、首部格式、校验 |
| TCP | 特点、首部格式、连接管理 |
| TCP可靠传输 | 序号、确认、重传 |
| TCP流量控制 | 滑动窗口 |
| TCP拥塞控制 | 慢开始、拥塞避免、快重传、快恢复 |

**TCP三次握手**：
```
Client → Server: SYN=1, seq=x
Server → Client: SYN=1, ACK=1, seq=y, ack=x+1
Client → Server: ACK=1, seq=x+1, ack=y+1
```

**TCP四次挥手**：
```
Client → Server: FIN=1, seq=u
Server → Client: ACK=1, seq=v, ack=u+1
Server → Client: FIN=1, ACK=1, seq=w, ack=u+1
Client → Server: ACK=1, seq=u+1, ack=w+1
```

**TCP拥塞控制**：
```
cwnd = 拥塞窗口
ssthresh = 慢开始门限

慢开始：cwnd从1开始，每RTT翻倍
拥塞避免：cwnd每RTT加1
快重传：收到3个重复ACK立即重传
快恢复：ssthresh = cwnd/2，cwnd = ssthresh
```

## 核心知识网络

```
体系结构概述
      ↓
物理层 ── 信道容量
      ↓
数据链路层 ─┬─ 差错控制(CRC)
            ├─ 流量控制(滑动窗口)
            └─ MAC协议(CSMA/CD)
      ↓
网络层 ─┬─ IP地址/子网划分
        ├─ 路由算法(RIP/OSPF)
        └─ IP分片
      ↓
传输层 ─┬─ TCP连接管理
        └─ 拥塞控制
      ↓
应用层 ── DNS/HTTP
```

## 学习方法建议

### 计算机网络学习特点

1. **协议多**：需要分层理解
2. **与实际结合**：可以抓包分析
3. **计算题固定**：子网划分、CRC等
4. **概念易混淆**：注意区分类似概念

### 学习建议

```
学习顺序：
1. 先理解分层模型
2. 每层从功能入手
3. 重点协议深入学习
4. 做题巩固计算
```
